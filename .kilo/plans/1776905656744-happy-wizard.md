# Plan: Fix OPENAI_API_KEY Tracing Error in spar.py

## Problem
The `agents` SDK (openai-agents>=0.13.0) has built-in tracing that attempts to export traces to OpenAI's API. When it tries to do this, it uses `OPENAI_API_KEY` from the environment, but the project only has `OPENROUTER_API_KEY` set. This causes a 401 error since the OpenRouter key isn't valid for OpenAI's tracing endpoint.

## Root Cause
The `agents` SDK automatically initializes tracing on import. When `Runner.run_streamed()` or similar methods are called, it attempts to export traces. The SDK checks for `OPENAI_API_KEY` to send traces to OpenAI's API, but:
1. The key being used is actually a valid OpenRouter key
2. OpenRouter doesn't support OpenAI's tracing endpoint
3. The SDK receives a 401 and logs the error

## Solution
Disable tracing programmatically at the start of the application by calling `set_tracing_disabled(True)` from the `agents.tracing` module.

### Changes to spar.py
Add after the existing dotenv import and before any agent initialization:

```python
from agents.tracing import set_tracing_disabled
set_tracing_disabled(True)
```

**Location**: After line 39 (`load_dotenv()`) and before line 40 (agents SDK imports).

### Why this works
- `set_tracing_disabled(True)` tells the SDK to skip tracing entirely
- The sparring functionality doesn't depend on tracing
- This eliminates the spurious 401 error without losing any features
