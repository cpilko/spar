# Plan: Fix Hanging Issue and Add Logging to spar.py

## Issues Identified

### 1. Debugging Visibility
The code has no visibility into what's happening before/after API calls. When the first call hangs, there's no output to indicate what's going on.

### 2. The Hang Location (based on traceback)
- "Round 1" header printed successfully
- Script hung inside `ask_agent(EMBER, ...)` before any events were yielded
- The `stream.stream_events()` was waiting on `self._event_queue.get()` when interrupted
- This means `Runner.run_streamed()` started but produced no events before the hang

### 3. Root Cause Hypothesis
The `Runner.run_streamed()` creates the agent and starts the run, but if the model or API returns nothing (or hangs before first event), we see no output. The current logging only prints when tool calls are detected inside `raw_parsed`, but if the initial model response doesn't yield any events, nothing is printed.
